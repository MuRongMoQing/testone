# 仓储存取

该上下文记录货物进入仓库、保持在库以及被取出的生命周期，并明确谁执行了状态变化。

## Language

### 架构

**模块化单体（Modular Monolith）**：
按业务能力和依赖方向拆分为多个可独立测试的 C++ 模块、但仍作为一个服务进程和一个 MySQL 数据库部署的架构；模块边界不等于网络服务边界。
_Avoid_: 单文件应用、微服务、仅按技术文件类型分组

**端口与适配器（Ports and Adapters）**：
领域与应用层定义所需数据库、汇率、附件扫描、通知和账册导出能力的接口，基础设施层提供 MySQL 或外部服务实现，使核心业务不依赖供应商 API 或传输细节。
_Avoid_: 全局数据库连接、领域层调用供应商 SDK、工具函数集合

**HTTP/JSON 适配层（HTTP/JSON Adapter）**：
使用 `cpp-httplib` 解析 HTTP 与 multipart、使用 `nlohmann/json` 解析和生成 JSON 的最外层协议模块；第三方库类型在此转换为自有 DTO 和应用命令，不进入领域或应用接口。
_Avoid_: 手写 HTTP 报文解析、正则 JSON 解析、领域对象直接序列化

**应用工作单元（Application Unit of Work）**：
一个业务命令使用的事务边界，统一锁定并原子写入申请、预留、批次、流水、余额、分录、审计和发件箱；提交或回滚由应用服务控制，HTTP 路由和单个仓储实现不能自行提前提交。
_Avoid_: 数据库连接、单表仓储、分布式事务

**命令事务（Command Transaction）**：
使用 `READ COMMITTED`、通过显式锁定读取保护将要改变的不变量，并按申请、预留、余额、批次的固定顺序加锁的短时读写工作单元。
_Avoid_: 普通查询、定稿快照、全局串行化

**定稿读取事务（Finalized Read Transaction）**：
使用 `REPEATABLE READ, READ ONLY, WITH CONSISTENT SNAPSHOT` 生成定稿报表、账册导出批次或审计校验的一致性只读工作单元；它必须有执行时限，不能跨 HTTP 请求长期保持。
_Avoid_: 实时列表、写命令、可更新游标

**数据库连接租约（Database Connection Lease）**：
一个工作单元从有界连接池独占取得、并在结束后归还的 RAII 连接所有权；同一连接不会被多个线程或并行业务命令同时使用。
_Avoid_: 全局连接、永久线程连接、数据库事务

**参数化 SQL（Parameterized SQL）**：
通过 MySQL 预处理语句绑定业务数据的查询或写入；值不得拼接进 SQL，动态标识符和排序只能取自代码允许列表，版本化迁移中的可信固定 SQL 不属于业务参数化接口。
_Avoid_: 字符串转义拼接、ORM、迁移脚本

**可重入分阶段迁移（Restartable Phased Migration）**：
把数据库演进拆为预检、单条原子 DDL、事务化 DML 回填、切换和后续清理，并为每一步记录可重复检查的完成状态；失败时拒绝业务启动并保留真实失败点，不能声称 MySQL 会把包含多条 DDL 的整个版本作为普通事务回滚。
_Avoid_: 整版 DDL 事务回滚、失败后继续启动、猜测式补偿

**版本化 API（Versioned API）**：
计划中的新业务接口统一位于 `/api/v1`；当前旧接口在对应新用例、迁移和对账获批前保持冻结兼容，不向旧路径增加新能力。
_Avoid_: 已实现接口、原路径破坏性替换、请求头版本猜测

**稳定键游标（Stable-Key Cursor）**：
普通列表按不可变或单调稳定键继续读取的位置标识；默认每页 50 条、最多 200 条，响应提供 `items` 和 `nextCursor`，避免并发新增导致页码重复或遗漏。
_Avoid_: 页码、账册顺序游标、数据库连接游标

**统一 API 错误（Uniform API Error）**：
计划中的 `/api/v1` 错误对象，固定包含 `code`、`message`、`details` 和 `requestId`，用于区分机器可处理错误、用户说明、结构化细节和请求追踪。
_Avoid_: HTTP 状态码、日志记录、成功响应

**Vue 业务前端（Vue Business Frontend）**：
计划采用 Vue 3、Vite、TypeScript、Vue Router 和 Pinia 构建的六身份浏览器工作台，并由 Vitest 与 Playwright 验证；前端可见性不能替代后端授权。
_Avoid_: 当前静态页面、权限边界、后端 API

### 身份与授权

**身份（Role）**：
分配给用户的固定职责类别，包括仓库员工、管理员、经理、监察员、会计和代理商；身份之间不存在统一的高低继承关系。
_Avoid_: 权限等级、职级

**单一身份账户（Single-Role Account）**：
同一时刻只拥有一种身份的用户账户；职责范围、临时代管和权限暂停作为独立状态存在，身份变更会保留历史并使旧会话失效。
_Avoid_: 多角色账户、临时代管身份

**高权限账户变更（Privileged Account Change）**：
增加、停用或改变经理、监察员、管理员或会计职责范围的治理动作，必须遵守相应双人控制、组织人数下限和永久审计要求。
_Avoid_: 普通员工管理、静默改库

**未分配账户（Unassigned Account）**：
已经拥有管理员或仓库员工身份，但尚未获得必要品类、上级、搭档或货物范围的账户；只能完成账户设置和查看通知，不能访问业务数据。
_Avoid_: 已停用账户、待审批申请

**双因素认证（Two-Factor Authentication）**：
密码之外使用 TOTP 验证登录者的安全机制；管理员、经理、监察员和会计必须启用，仓库员工与代理商可以选择启用。
_Avoid_: 两个密码、安全问题

**第二因素恢复（Second-Factor Recovery）**：
用户丢失 TOTP 或恢复码后重新建立第二因素的受控流程；高权限账户必须由匹配其身份的双人流程完成。
_Avoid_: 单人重置、关闭双因素认证

**数据库浏览器会话（Database-Backed Browser Session）**：
由 256 位随机不透明令牌标识、只在 MySQL 中保存令牌哈希的登录会话；浏览器通过 `Secure`、`HttpOnly`、`SameSite=Strict` 的主机限定 Cookie 携带，服务端执行闲置与绝对超时。
_Avoid_: 内存 Token、JWT、localStorage 凭据

**敏感操作重新认证（Step-Up Reauthentication）**：
高权限账户执行账户治理、权限恢复、临时代管、仓库结构变更、重大会计决定或外部账册映射前，重新验证 TOTP 并只在随后 10 分钟内认可该验证的机制。
_Avoid_: 再次登录、永久提升、普通库存查询

**CSRF 防护令牌（CSRF Protection Token）**：
与浏览器会话绑定、由前端在状态变更请求的自定义请求头中显式提交的不可预测值；服务端还校验 Origin 或 Fetch Metadata，不能只依赖 SameSite Cookie。
_Avoid_: 会话 Cookie、幂等键、URL 参数

**服务账户（Service Account）**：
供外部总账或上下级仓库账册调用集成接口的不可交互机器身份；它不属于六类人员身份，不具备网页登录、密码或 TOTP，只能在获批的仓库子树、品类和接口动作范围内使用可轮换、可立即撤销的独立服务凭据。
_Avoid_: 用户账户、浏览器会话、共享 API 密钥

**账册事件签名（Ledger Event Signature）**：
推送账册事件时以 HMAC-SHA256 覆盖事件 ID、时间戳和完整请求正文生成的完整性证明；接收方按正负 2 分钟时间窗口和事件唯一性拒绝重放。服务证书和 HMAC 密钥最长有效 30 天，并允许新旧凭据重叠 12 小时完成轮换。
_Avoid_: 审计哈希链、附件哈希、TLS 证书

**挂载式密钥材料（Mounted Secret Material）**：
由部署环境以受限文件挂载方式提供的 TOTP 主加密密钥、TLS 私钥或 HMAC 密钥；应用配置只引用文件路径，密钥值不得写入数据库、源码、日志或审计载荷。
_Avoid_: 数据库凭据指纹、环境变量中的明文密钥、浏览器会话

**权限（Permission）**：
允许用户执行一项明确业务动作的授权；每种身份拥有一组显式权限。
_Avoid_: 身份、角色等级

**责任范围（Responsibility Scope）**：
一个用户获准查看和操作的货物集合；管理员范围是一个品类与一个仓库子树的交集，仓库员工范围是其绑定仓库内获分配的货物。经理、监察员和总会计不受仓库子树限制。
_Avoid_: 全局权限、身份等级

**管理员范围分配（Administrator Scope Assignment）**：
将一名管理员绑定到恰好一个货物品类和一个仓库子树根节点的有效期记录；同一品类的有效管理员范围不得在仓库树上重叠。初始部署时所有管理员绑定唯一根仓库，新增仓库并完成组织分配后即可在互不重叠的子树间拆分责任范围。
_Avoid_: 多品类身份、重叠审批人、临时代管

**员工搭档（Staff Pair）**：
共同负责同一组货物的两名仓库员工，其中一人的操作会通知另一人。
_Avoid_: 上下级、审批人

**提醒（Notification）**：
用户下次进入系统时以弹窗显示的业务信息，用于告知配置变化、搭档操作及其他需要关注的事件。
_Avoid_: 审批、警告、实时消息

**警告（Warning）**：
通过与提醒相同的弹窗方式送达、表示规则违反或风险升级的信息。
_Avoid_: 提醒、审批拒绝

**审计事件（Audit Event）**：
与业务变更在同一事务中追加、记录操作者、当时身份与范围、动作、目标、原因、前后摘要及请求环境的不可修改事实。
_Avoid_: 可编辑日志、业务当前状态

**审计哈希链（Audit Hash Chain）**：
所有审计事件使用长度前缀规范编码和 SHA-256 形成一条全局连续校验链；追加事件时锁定唯一链头行，并将前一事件哈希纳入新事件哈希，用于检测历史事件被删除、插入或改写。
_Avoid_: 加密存储、数字签名

**幂等业务命令（Idempotent Business Command）**：
由唯一幂等键标识的一次预期审批或库存执行；同一键和相同内容被重复提交时返回首次结果而不再次改变业务状态，同一键对应不同内容时拒绝处理。
_Avoid_: 数据库行号、允许重复执行的 HTTP 请求、去重提示

### 库存

**仓库（Warehouse）**：
承担库存保管责任的业务节点，具有永久唯一编码，并可通过上级仓库形成总仓、区域仓和分仓等组织层级；当前遗留实现尚未建模仓库，首版目标数据库、后端接口和业务流程将完整支持多个仓库，初始目标部署只创建一个根仓库。
_Avoid_: 货位、管理员品类、单仓全局常量

**多仓业务启用条件（Multi-Warehouse Enablement Condition）**：
目标系统存在至少两个有效仓库，且相关仓库已经完成管理员范围、员工、货位及货物职责分配的状态；达到条件后前端显示跨仓调拨入口，未达到时只隐藏入口而不改变后端多仓接口。当前遗留实现不具备该能力。
_Avoid_: 功能占位、数据库迁移开关、绕过组织分配

**仓库组织层级（Warehouse Hierarchy）**：
通过仓库之间的父子关系表达总仓、区域仓和分仓结构的树；它描述保管责任节点，不替代仓库内部的物理位置层级。
_Avoid_: 货位层级、品类层级、权限等级

**货位层级（Location Hierarchy）**：
一个仓库内部由库区、通道、货架到叶级货位组成的树；库存余额、预留和实际操作只落在可用叶节点，父节点用于导航、汇总与权限接口。
_Avoid_: 仓库组织层级、自由文本货位、地址

**品类（Goods Category）**：
用于划分货物管理责任的一组货物品种；同一仓库子树范围内由一名管理员负责，同一品类可由多名管理员分别负责互不重叠的仓库子树，并由经理跨品类、跨仓库统筹。
_Avoid_: 货物品种、货位

**计量单位（Unit of Measure）**：
货物品种计算库存数量所使用的单位，例如件、箱、千克或升。
_Avoid_: 包装说明、价格单位

**数量精度（Quantity Precision）**：
一种货物数量允许保留的小数位数，范围为 0 至 6；数量以 `DECIMAL(38,6)` 保存，每次存取数量必须大于零并符合该 SKU 精度，取出量还不得超过可用库存，超出存储范围时拒绝处理。
_Avoid_: 浮点误差、显示格式

**货物编码（SKU）**：
货物品种跨批次保持不变的唯一业务身份，也是前端汇总和判断全新货物的依据；已归档 SKU 再次入库时恢复原品种和历史，不产生新 SKU。
_Avoid_: 货物名称、批次编号、数据库行号

**期初成本待确认（Opening Cost Pending）**：
旧系统在库记录迁移后因缺少历史进价而无法完成库存估值的状态；经会计组三人中至少两人同意并由经理批准前，不得取出或进入定稿财务报表。
_Avoid_: 零成本、免费库存、已估值

**货物品种（Goods Type）**：
由唯一 SKU 标识并具有名称、规格、品类、计量单位和数量精度的货物定义，也是汇总展示和新货审批的对象。
_Avoid_: 货物记录、商品、库存行

**全新货物提案（New Goods Proposal）**：
为尚未存在的 SKU 提交的货物品种及首次入库提议；提交时即以待审批状态占用 SKU，但不创建库存余额且不进入在用货物目录。批准后等待首次入库，只有实际执行入库才转为在用；撤回或最终拒绝后仍永久留档，已占用 SKU 不得分配给其他货物。
_Avoid_: 在用货物品种、库存批次、可复用 SKU

**库存批次（Stock Lot）**：
同一种货物在一次入库中形成的数量集合；不同批次保持彼此可区分，但默认库存视图会按货物品种汇总。
_Avoid_: 货物、库存余额

**库存余额（Inventory Balance）**：
一个 SKU 在一个仓库的一个叶级货位当前仍由仓库保管的数量投影，由该货位所有有效库存批次叠加而成；用户不能直接编辑。
_Avoid_: 仓储数表单、库存表单、库存批次

**汇总库存（Aggregated Stock）**：
一个 SKU 在所有货位的库存余额之和，是前端默认展示的数量；展开后显示各货位余额和相关批次。
_Avoid_: 货位余额、单个库存批次

**预留库存（Reserved Stock）**：
审批人在批准取出、销售取出或库内移位申请时，为尚未实际执行部分确定并占用的来源货位数量；预留不减少在库数量，目标货位也不提前增加，但其他申请不能重复占用。员工可以提出非约束性的建议来源货位，代理商不能填写或查看来源货位；每次分批执行消耗对应预留，期限届满时释放剩余预留。
_Avoid_: 已取出库存、可用库存

**可用库存（Available Stock）**：
在库数量减去预留库存后的数量，是新取出申请可以占用的上限。
_Avoid_: 在库数量、预留库存

**预留期限（Reservation Period）**：
取出或库内移位申请获批后允许执行的时间窗口，默认 24 小时并可配置；到期前 2 小时提醒相关人员，到期时释放尚未执行部分的来源库存，剩余数量如仍需操作必须重新申请。
_Avoid_: 补货宽限期、审批超时

**执行期限（Execution Period）**：
仓库员工的入库、取出或库内移位申请获批后允许实际执行的时间窗口，默认 24 小时；到期前 2 小时提醒相关人员，到期时取消尚未执行的剩余数量，剩余业务必须重新提交申请。
_Avoid_: 审批超时、补货宽限期

**分批执行（Partial Execution）**：
在执行期限内将一份获批申请分为一次或多次实际操作；每次数量必须大于零，累计数量不得超过批准数量，并分别形成不可变的执行结果、库存流水、批次分配及适用的会计分录。累计数量未达到批准数量时，申请处于部分执行状态。
_Avoid_: 修改批准数量、覆盖执行结果、重复执行

**库存流水（Stock Movement）**：
一次已经发生的库存数量或货位变化，是追溯库存余额及相关财务事实的长期记录。
_Avoid_: 库存余额、操作日志

**业务单据（Supporting Document）**：
证明一次入库、取出或库内移位业务依据的必需附件，只接受 JPEG、PNG、WebP、PDF 或 DOCX；拒绝加密文件、旧版 DOC、含宏 DOCM、SVG、可执行文件和压缩包，关联业务记录后不可覆盖。
_Avoid_: 库存流水、备注、可选附件

**隔离附件（Quarantined Attachment）**：
已经上传但尚未通过格式验证或恶意软件扫描，或者已被判定存在风险，因而不能作为有效业务单据使用的附件版本。
_Avoid_: 已批准单据、已删除文件

**附件安全审查（Attachment Security Review）**：
对隔离附件执行格式验证和恶意软件扫描并记录引擎、签名、时间与结论的过程；只有明确判定安全的附件才能进入业务流程。
_Avoid_: 人工审批、文件扩展名检查

**附件版本（Attachment Version）**：
业务单据或代理合同被替换时新增的不可变文件版本，与先前版本共同保留并可追溯上传者和上传时间。
_Avoid_: 覆盖文件、临时缓存

**库存操作申请（Stock Movement Request）**：
一笔只针对一个 SKU、尚待审核的入库、取出或库内移位提议，包含该操作所需的数量、价格或货位以及业务单据；它本身不是已经发生的库存流水。同一业务单据包含多个 SKU 时，为每个 SKU 建立独立申请并复用同一附件版本。
_Avoid_: 库存流水、库存余额、草稿

**公共申请持久化结构（Common Request Persistence Structure）**：
目标数据库以 `stock_operation_requests` 保存申请类型、申请人、状态、审批超时、执行期限和复核链，以 `inbound_request_details`、`outbound_request_details`、`transfer_request_details` 分别保存三种类型明细；审批决定、复核次数、附件、预留和执行结果使用独立关联表。申请与批准不写入库存事实，只有实际执行才原子写入流水、批次分配、余额和会计分录。
_Avoid_: 当前数据库表、单一申请巨表、执行流水

**库存操作审批（Stock Movement Approval）**：
管理员或经理对单笔库存操作申请作出的批准或拒绝决定；仓库员工提交的每笔入库、取出和库内移位都必须单独审批。
_Avoid_: 货物品种审批、批量授权

**申请撤回（Request Withdrawal）**：
申请人在最终批准或拒绝前主动关闭自己申请的动作；撤回保留申请、附件和审计历史，不产生库存或会计事实。全新货物提案撤回后，其已经占用的 SKU 仍不得复用。
_Avoid_: 删除申请、拒绝、获批后取消

**剩余量取消申请（Remainder Cancellation Request）**：
原申请获批后，请求取消全部尚未执行数量或部分执行后的剩余数量的独立申请；它沿用原申请的管理员优先、经理超时接管审批流程。获批只释放剩余预留并关闭余量，已经执行的库存与会计事实保持不变。
_Avoid_: 申请撤回、冲正、修改批准数量

**库内移位（Internal Transfer）**：
在同一仓库内将指定 SKU 的数量及其原批次成本从一个叶级货位转到另一个叶级货位的库存动作；它产生移位流水，但不改变该仓数量、库存总价值、收入或利润。
_Avoid_: 取出后重新入库、销售、采购

**跨仓调拨（Inter-Warehouse Transfer）**：
由发出仓内负责该 SKU 的仓库员工申请、在不同仓库责任节点之间移动库存的独立流程；发出仓管理员批准数量和来源预留，接收仓管理员批准接收及目标叶级货位，两边均批准后才可分批发运和收货。若发出仓管理员暂停，发出侧审批转交经理；若同一管理员覆盖两端，接收侧由经理作为不同人员完成第二审批。
_Avoid_: 库内移位、销售取出、目的货位即时增加

**在途库存（In-Transit Inventory）**：
跨仓调拨已经由发出仓实际发运、但尚未由接收仓实际收货的数量及原批次成本；它不再计入发出仓可用库存，也不提前计入接收仓货位余额。发运时从发出仓库存资产转入在途库存资产，收货时再转入接收仓库存资产，始终沿用原批次原币成本、锁定汇率和人民币成本。
_Avoid_: 预留库存、接收仓库存、已损耗库存

**调拨差异（Transfer Discrepancy）**：
跨仓调拨的累计实收数量与对应在途数量不一致时形成的待调查记录；发出与接收两侧负责人分别提交说明和证据。短少继续保留为在途库存，超收进入隔离库存，系统不自动调整、丢弃或视为损耗。
_Avoid_: 自动盘亏、库存删除、会计冲正

**调拨隔离库存（Transfer Quarantine Inventory）**：
接收仓实际发现但超过对应在途数量、来源尚未确认的实物保管记录；它不可销售、移位或作为接收仓库存资产入账，必须关联正确调拨或另行通过正常入库审批后才能转为可用库存。
_Avoid_: 可用库存、在途库存、盘盈自动入账

**季度抽样复核（Quarterly Sample Review）**：
监察员按 `Asia/Shanghai` 自然季度抽取管理员经办记录：缺单、逾期和已有异常记录全部必选，再加入人民币绝对金额最高 5% 的记录，并使用永久保存的抽样种子随机补足到当季记录的 10%。常规样本至少 10 条、最多 100 条，不足 10 条时全部检查，风险必选项不受 100 条上限；任一异常触发当季全量复核。
_Avoid_: 逐笔预审、全量审计

**季度全量复核（Quarterly Full Review）**：
季度抽样发现异常后，监察员对该管理员当季全部库存记录进行的完整核对。
_Avoid_: 常规抽样、永久监控

**强制预审期（Mandatory Preapproval Period）**：
经理指定的一段时间，在此期间管理员提交的库存操作必须经经理批准后才能生效。
_Avoid_: 权限暂停、季度抽样复核

**管理员权限暂停（Administrator Suspension）**：
管理员因季度单据逾期或复核异常而被临时禁止审批及其他高风险写操作的状态；该管理员仍可登录、只读查看、补交单据和回应调查，直至经理发起恢复且监察员批准。
_Avoid_: 停用账户、解雇、强制预审期

**范围配置覆盖（Scoped Configuration Override）**：
管理员在其“品类与仓库子树”职责交集内对补货宽限期等经理全局默认值设置的覆盖；它不得影响其他管理员范围，变更进入审计并提醒经理。
_Avoid_: 全局配置、静默修改、永久品类归属

**管理员季度后补单（Administrator Quarterly Document Deferral）**：
管理员默认后审操作可以先形成申请、执行、库存和会计事实，并在当季结束前补交单据；暂时缺少附件是该流程的既定例外，但上传文件只有通过格式验证和恶意软件扫描后才算补齐，不安全或扫描异常文件永不视为有效单据。
_Avoid_: 员工免交单据、不安全附件放行、覆盖历史附件

**临时代管（Temporary Delegation）**：
经理在监察员同意后，将明确货物范围内的必要管理权限限时授予另一名管理员；它不改变货物品类的永久归属。
_Avoid_: 永久调岗、品类转移、无限授权

**管理变更回退（Administrative Change Reversal）**：
监察员撤销经理对管理员账户或品类分配所作管理变更的治理动作；它不删除变更期间形成的审计记录。
_Avoid_: 删除历史、库存冲正

**审批超时（Approval Timeout）**：
仓库员工申请提交满一小时仍未被对应管理员批准或拒绝的状态；仅查看、打开或标记处理中不算完成处理，超时后经理也可作出决定。
_Avoid_: 已读、处理中、管理员拒绝

**复核申请（Reconsideration Request）**：
仓库员工修改被拒绝的入库申请后再次请求审批的新不可变版本；它通过根申请和上一版本形成复核链，沿用正常审批流程，并保持原申请、原附件、原审批结果及原审批员不变。同一复核链连续三次复核均被拒绝后关闭复核入口。
_Avoid_: 覆盖原申请、修改历史审批、无关联的新申请、申诉

### 会计

**先进先出（FIFO）**：
取出货物时优先消耗最早仍有余额的库存批次；一次取出可以跨批次，并按各批次实际进价计算销售成本。
_Avoid_: 后进先出、移动平均、员工手选成本

**销售收入（Sales Revenue）**：
一次取出的数量与其单位售价相乘所得金额。
_Avoid_: 毛利润、现金收入

**销售成本（Cost of Goods Sold）**：
一次取出按照先进先出实际消耗的各库存批次数量与对应单位进价计算所得成本。
_Avoid_: 销售收入、库存估值

**毛利润（Gross Profit）**：
销售收入减去销售成本后的金额。
_Avoid_: 销售收入、净利润

**记账本位币（Base Currency）**：
统一账册汇总金额使用的人民币（CNY）；日元、美元和欧元交易在保留原币记录的同时折算为人民币。
_Avoid_: 交易币种、显示币种

**交易币种（Transaction Currency）**：
一笔业务实际计价使用的 CNY、JPY、USD 或 EUR；外币交易进入对应原币账册。
_Avoid_: 记账本位币、汇率

**不含税价格（Tax-Exclusive Price）**：
库存子账用于计算库存成本、销售收入和毛利润的单位进价或售价，不包含税额；含税业务单据可以留档，但首版不计算或申报税款。
_Avoid_: 含税总价、税额

**金额精度（Monetary Precision）**：
单位进价和售价使用 `DECIMAL(38,4)` 且必须大于零，汇率使用 `DECIMAL(38,8)`；CNY、USD、EUR 的原币总额及人民币分录使用 `DECIMAL(38,2)`，JPY 输入必须为整数金额，所有金额统一四舍五入并在溢出时拒绝处理。
_Avoid_: 浮点金额、显示精度

**汇兑舍入差额（FX Rounding Difference）**：
外币金额折算及分录逐行舍入后，为保持借贷平衡而单独记录的微小人民币差额。
_Avoid_: 汇兑损益、手工抹零

**锁定汇率（Locked Exchange Rate）**：
一笔外币业务在库存与账册实际生效时用于折算人民币的汇率事实，与原币金额、人民币金额、来源、报价时间和系统取得时间共同永久保存，之后不会随市场汇率重算；首版只接受报价时间不超过 90 分钟的数据。
_Avoid_: 当前显示汇率、历史重估

**外币账册（Foreign-Currency Ledger）**：
按 JPY、USD 或 EUR 分别记录原币金额的账册，并为每笔记录关联锁定汇率及折算人民币金额。
_Avoid_: 人民币统一账册、汇率缓存

**复式库存会计子账（Double-Entry Inventory Subledger）**：
围绕入库、取出、跨仓在途库存、库存资产、销售收入、销售成本和毛利润自动生成借贷平衡记录的账册；首版不处理实际收付款、应收应付、税务、工资或费用。
_Avoid_: 完整企业总账、库存流水

**账册导出事件（Ledger Export Event）**：
与内部会计分录在同一事务中写入事务性发件箱、供总账或上下级仓库账册幂等拉取或接收的不可变消息；外部确认和凭证号另行追加，不改变原分录。
_Avoid_: 外部回写、可删除消息、会计分录副本

**仓库层级路径快照（Warehouse Hierarchy Path Snapshot）**：
会计分录入账时保存的仓库祖先路径及编码快照，用于让上下级仓库账册的历史归属不因日后仓库重组而改变。
_Avoid_: 当前仓库树、权限范围、货位路径

**外部账册映射（External Ledger Mapping）**：
将内部科目、仓库及币种等维度转换为特定总账或仓库账册编码的版本化配置；改变映射可能改变外部账册含义，因此属于重大会计决定。
_Avoid_: 直接改分录、供应商专用领域模型、显示别名

**会计分录（Journal Entry）**：
由已经生效的库存业务或获批会计调整产生、借贷总额相等且不可覆盖的财务事实；更正只能通过关联的反向或补充分录完成。
_Avoid_: 库存流水、直接改账

**采购暂记（Purchase Clearing）**：
首版库存子账在入库时用于平衡库存资产借方的过渡科目，不代表实际应付账款或现金支付。
_Avoid_: 应付账款、银行存款

**销售暂记（Sales Clearing）**：
首版库存子账在确认销售收入时使用的过渡借方科目，不代表实际应收账款或现金收款。
_Avoid_: 应收账款、银行存款

**品类会计组（Category Accounting Group）**：
由三名会计组成、负责一个或两个货物品类财务工作的固定小组；任何有效小组不得少于三名成员或负责超过两个品类。
_Avoid_: 单人会计、仓库员工组

**总会计组（Chief Accounting Group）**：
由三名总会计组成、对所有货物品类财务工作负总责的小组。
_Avoid_: 经理、单一总会计

**会计调整申请（Accounting Adjustment Request）**：
会计针对错误财务记录提出的非覆盖式更正请求，必须说明原因并附单据；原记录永久保留，申请须经经理批准并提醒监察员。
_Avoid_: 直接改账、删除原记录

**普通会计操作（Routine Accounting Action）**：
不会改变已入账金额、会计期间状态、计价规则或定稿报表的日常核对、说明、差异标记、未定稿报表生成和数据导出；一名会计即可执行并提醒同组另外两人。
_Avoid_: 重大会计决定、库存操作

**重大会计决定（Material Accounting Decision）**：
会改变已入账金额、关闭或重开会计期间、改变成本与估值、利润确认或外部账册映射规则，或者撤销、替换定稿财务报表的决定；必须获得组三人中至少两人同意。
_Avoid_: 普通会计操作、管理员配置

### 代理关系

**代理合同（Agency Contract）**：
证明代理商身份及授权范围的必需 PDF 或 Word 文档，由经理增加或取消代理商身份时提交并留档。
_Avoid_: 库存业务单据、口头授权、可选附件

**代理商复核异议（Agent Review Objection）**：
监察员对经理增加或取消代理商身份所提出的复核请求；它会启动复核但不能由监察员直接回退经理的决定。
_Avoid_: 管理变更回退、否决

**销售取出申请（Sales Withdrawal Request）**：
代理商针对获分配货物提交的数量、售价和业务单据请求，由所属品类管理员优先审批，超时或管理员暂停时由经理处理；审批人在代理商不可见的库存预留中确定来源货位，获批后由负责该货物的仓库员工执行。
_Avoid_: 库存流水、采购入库、代理商直接取货

**入库（Store）**：
增加一种货物的库存数量并形成库存批次的业务动作。
_Avoid_: 新增、上架

**在库（Stored）**：
货物品种的库存余额大于零、仍可被取出的状态。
_Avoid_: 可用、活动

**取出（Take）**：
减少一种货物的库存数量并记录操作员的业务动作。
_Avoid_: 删除、出库

**耗尽批次（Depleted Stock Lot）**：
剩余数量已经降至零的库存批次；它仍是库存和财务历史的一部分。
_Avoid_: 已删除、已归档、无库存

**待归档（Pending Archive）**：
SKU 在所有货位的实际在库总量因已执行取出而归零后、补货宽限期结束前的状态；预留或未执行入库不影响计时，只有已执行入库使总量重新大于零才恢复在库，再次归零时重新开始完整宽限期。
_Avoid_: 准移除、已删除、已取出

**已归档货物品种（Archived Goods Type）**：
补货宽限期内始终保持零库存、因而不再出现在在用目录中的货物品种，其既有批次、流水和财务历史仍被保留。
_Avoid_: 已删除货物、已取出货物

**货位（Location）**：
仓库物理位置树中的节点，具有永久唯一且不可修改的编码、可修改的名称、节点类型和状态；库存只能落在可用叶级货位，同一 SKU 可以分布在多个货位。停用不删除历史，存在库存、有效预留或已批准待执行移入时不得停用。
_Avoid_: 地址、自由文本、仓库

**旧货待分配货位（Legacy Unassigned Location）**：
根仓库中用于承接全部旧 `goods.location` 自由文本记录的受控叶级货位；迁移不把自由文本猜测为正式位置，原始文本永久保存在旧货映射中，之后由经理通过审计式流程分配正式货位。
_Avoid_: 自动按文本建货位、丢弃原位置、默认货架

**操作员（Operator）**：
执行一次入库、取出或其他库存变化的已认证用户。
_Avoid_: 所有人、管理员

**补货宽限期（Restock Grace Period）**：
货物品种从库存归零到自动归档之间、允许通过补货恢复在库的时间窗口；经理维护默认 30 天的全局值，管理员只能在自己的“品类与仓库子树”范围内设置覆盖值并提醒经理。
_Avoid_: 保留期、缓存期、删除倒计时
