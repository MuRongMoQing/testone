## 当前事实与目标边界

阶段 1 已把编译路径重构为 C++17 模块化单体：`main` 进入组合根，HTTP/JSON、typed 应用用例、MySQL RAII/工作单元、版本化迁移和启动生命周期分属独立目标；旧单文件实现只作为未编译参考保留。当前仍只有遗留 `users`/`goods` 业务表、内存 Bearer Token、三级角色和静态原生前端，不存在 `/api/v1`、六身份或新库存业务能力。Debug 与 Release 的完整 CTest 各 14 项均已运行通过，其中第 14 项在专用本地 MySQL 8 测试库上真实执行；这只证明阶段 1 的本地门禁，不代表 CI、生产迁移或后续阶段能力。

目标是一个 C++17 模块化单体和一个 MySQL 8 数据库，使用 `cpp-httplib`、`nlohmann/json`、RAII `libmysql`、libsodium、OpenSSL 3，以及 Vue 3 + Vite + TypeScript 前端。除非某阶段报告提供真实实现和验证证据，目标能力不得描述为当前能力。

## 锁定业务契约

### 库存与目录

- SKU 是跨批次唯一身份；一次申请只针对一个 SKU。
- `inventory_balance` 的维度是仓库、叶级货位和 SKU，是可重建、不可人工编辑的事务投影。
- 默认前端按 SKU 汇总，展开仓库、货位和批次。
- 数量使用 `DECIMAL(38,6)`，SKU 精度为 0 至 6；数量必须大于零且符合精度。
- 单位进价和售价使用 `DECIMAL(38,4)` 且必须大于零；汇率使用 `DECIMAL(38,8)`；总额使用 `DECIMAL(38,2)`，JPY 输入为整数。
- `available = on_hand - reserved`；审批预留不减少在库，实际执行才改变在库。
- 取出按 FIFO 跨批次分配成本；同仓移位保留原批次成本且不产生金额分录。
- 每次分批执行不可变，累计数量不得超过批准量；默认执行/预留期限 24 小时，提前 2 小时提醒，到期关闭余量并释放预留。
- 实际在库归零开始默认 30 天待归档；只有实际入库取消计时，再次归零重新完整计时。归档只移出在用目录，不删除批次、流水、分录或审计历史。

### 公共申请和新 SKU

- `stock_operation_requests` 保存公共头。
- `inbound_request_details`、`outbound_request_details`、`transfer_request_details` 保存类型明细。
- 审批、员工入库复核、附件、来源建议、预留、余量取消、幂等结果和执行结果使用独立关联表。
- 申请或批准本身不写库存事实；执行才在同一工作单元写入执行结果、流水、批次分配、余额、会计分录、审计和发件箱。
- 最终审批前申请人可撤回；批准后只能提交独立余量取消申请，不能覆盖已执行事实。
- 被拒的员工入库申请可以建立不可变复核版本；连续三次复核拒绝后关闭该链。该三次规则不泛化到代理商或其他申请类型。
- 审批和每次执行携带幂等键；同键同规范载荷返回原结果，同键不同载荷返回冲突。
- 新 SKU 提交时进入 `PENDING_APPROVAL` 并永久占用；批准后为 `APPROVED_AWAITING_FIRST_RECEIPT`，首次实际入库后为 `ACTIVE`。撤回或拒绝的编码永不复用。

### 身份与治理

- 人员账户同一时刻只能有仓库员工、管理员、经理、监察员、会计或代理商中的一种身份；服务账户不是人员身份。
- 权限不按等级继承，必须同时检查动作、品类、仓库子树、货物分配、临时代管和暂停状态。
- `PERMISSIONS.md` 中每一条允许、禁止、范围、治理和字段可见性规则都是规范性要求，所属阶段必须逐条建立 allow/deny/field-visibility 测试并在追踪结果中引用，不能只用身份名称的少量示例代替完整矩阵。
- 员工只能在绑定仓库操作分配货物，但可按员工字段范围查看上级管理员职责范围；搭档收到操作提醒。
- 员工每笔入库、取出、移位和新货申请逐笔审批并提供安全单据；员工不能审批、改历史、看完整账册/利润或治理系统。
- 管理员范围是一个品类与一个仓库子树的交集；同品类管理员范围不得重叠。
- 管理员默认后审操作仍形成完整申请、执行、流水、会计和审计事实，但不产生自己批准自己的决定；强制预审期由经理批准。
- 管理员可季度后补单；暂缺附件不阻止默认后审事实，但文件只有扫描安全后才算补齐。
- 管理员只能覆盖自身范围配置，经理维护全局默认；覆盖变更审计并提醒经理。
- 单据逾期或抽样异常暂停管理员写权限，保留只读、补单和整改；员工申请转经理。恢复由经理发起、监察员批准。
- 临时代管必须经监察员同意，限定货物范围和期限，不改变永久归属。
- 经理仅在员工/代理商申请满一小时仍无最终决定后接管；并发以首个合法提交为准。经理默认不亲自执行库存，也不能直接提交跨仓调拨。
- 监察员全局只读和治理审查，不执行库存或审批普通库存申请。
- 品类会计组三人，负责一个或两个品类；总会计组三人全局负责。普通操作一人执行并提醒另两人，重大决定至少两票；非全票永久保存议程并警告经理和监察员，已执行决定只能在二者均认定错误后追加回退。
- 代理商只看获分配商品的可售汇总，不能看成本、利润、精确货位或其他代理商数据，不能执行或审批出库。

### 季度复核

- 使用 `Asia/Shanghai` 自然季度。
- 缺单、逾期和已有异常记录全部必选。
- 加入人民币绝对金额最高 5% 的记录。
- 使用永久保存的种子随机补足到 10%。
- 常规至少 10 条、最多 100 条；不足 10 条全部检查；风险必选项不受上限。
- 保存规则版本、种子、候选集合、入选原因和结果；任一异常升级为当季全量复核。

### 附件与安全

- 二进制以 MySQL `LONGBLOB` 不可变版本保存，记录文件名、MIME、大小、SHA-256、上传者和时间；单文件上限 20 MiB。
- 允许 JPEG、PNG、WebP、PDF、DOCX；拒绝 DOC、DOCM、SVG、普通压缩包、可执行文件、加密文档和含宏文档。
- 文件先隔离并校验扩展名、MIME 和真实签名，再流式交给 ClamAV。
- 扫描风险、不可用、报错或病毒库超过 48 小时均失败关闭；保存扫描证据，不自动删除风险文件。
- 管理员季度后补只豁免当时缺少附件，不允许未扫描或异常文件成为有效单据。
- 生产依赖或发布二进制未扫描、扫描工具异常或结果异常时不得放行。

### 会计与汇率

- 首版只实现不含税的复式库存会计子账，不承担应收应付、现金、税务、工资或费用总账。
- 入库：借库存资产，贷采购暂记。
- 取出：借销售暂记、贷销售收入；借销售成本、贷库存资产。
- 同仓移位不产生金额分录；汇率舍入差额进入独立科目。
- 跨仓发运：借在途库存、贷发出仓库存；收货：借接收仓库存、贷在途库存。
- 支持 CNY、JPY、USD、EUR；原币与人民币同时保存，采用四舍五入，历史不随当前汇率重算。
- 汇率端口顺序为 ExchangeRate.fun、USDConverter、本地最后成功缓存；ECB 不作为自动后备。
- 报价不得超过 90 分钟，保存提供方、报价时间、获取时间、原币金额、汇率和人民币金额。
- 无合格汇率时整个外币库存和账务命令无副作用，CNY 业务不受影响。模拟器通过不能代替目标网络真实供应商验证。
- 错误账务只通过关联的反向或补充分录更正，原分录永久保留。

### 多仓与调拨

- `warehouses(parent_warehouse_id)` 是组织树；`storage_locations(warehouse_id,parent_location_id)` 是物理位置树，两者不能混用。
- 仓库和货位编码永久唯一且不可修改；库存只落在有效叶级货位。存在库存、预留或待执行移入时不得停用。
- 初始目标部署只有一个根仓库，但后端目标模型和 API 真实支持多仓；至少两个有效仓库且组织分配完整后才显示跨仓入口。
- 只有发出仓负责 SKU 的员工可以申请跨仓调拨；管理员和经理只能发送与申请关联的留痕通知。
- 发出管理员审批并预留，接收管理员指定目标叶位；两端必须由不同人员。相同管理员覆盖两端时由经理承担接收侧；发出管理员暂停时经理承担发出侧，接收侧仍需另一人。
- 发运后进入在途，收货后才增加接收仓；发运和收货均可分批且分别附单据。
- 原批次原币成本、锁定汇率和人民币成本全程不重算，不产生收入或利润。
- 短少保持在途，超收进入不可用、不入账的隔离库存。损坏或丢失需要会计组两票、经理批准并提醒监察员。

### 会话、审计和外部账册

- 浏览器令牌由 libsodium 生成 256 位随机值，数据库只存哈希；Cookie 为 `__Host-warehouse_session; Secure; HttpOnly; SameSite=Strict; Path=/` 且无 `Domain`。
- 写请求使用 `X-CSRF-Token` 并检查 Origin/Fetch Metadata，不把令牌放入 `localStorage`。
- 管理员、经理、监察员、会计闲置 15 分钟、绝对 8 小时；员工、代理商闲置 30 分钟、绝对 12 小时。
- 账户治理、管理员恢复、临时代管、仓库结构、重大会计决定和外部映射要求最近 10 分钟内 TOTP 重新认证。
- 登录、重新认证、身份或密码变化、TOTP 恢复和管理员恢复会轮换或撤销相关会话。
- TOTP 使用 OpenSSL 3 RFC 6238 HMAC-SHA1；libsodium XChaCha20-Poly1305 加密 TOTP 密钥。
- 审计事件与业务同事务追加；全局链使用长度前缀规范编码、SHA-256 和唯一链头行锁。应用账户只能查询/插入，数据库阻止更新/删除。
- 内部子账是事实源，分录和版本化 outbox 同事务；事件包含分录/行、来源、币种、人民币、汇率、仓库、入账路径快照、调整关联和完整性摘要。
- 上级账册按入账路径子树聚合，下级按具体仓库过滤，不复制分录；外部只能追加确认和引用号，错误回到内部调整流程。
- 服务账户按仓库子树、品类和动作最小授权，由经理发起、监察员批准；不得使用浏览器 Cookie 或修改内部事实。
- 服务证书和 HMAC 密钥最长 30 天，重叠轮换 12 小时；HMAC 覆盖事件 ID、时间戳和原始正文，时间窗口为正负 2 分钟并按事件 ID 防重放。
- TOTP 主密钥、TLS 私钥和 HMAC 密钥通过可替换端口从部署挂载文件读取，环境变量只保存路径。

### 保守迁移

- 旧 `admin` 和 `manager` 迁为未分配管理员，旧 `viewer` 迁为未分配员工；保留用户名、Argon2id 哈希和历史关联。
- 每条旧 `goods` 独立生成 `LEGACY-Gxxxxxx`，不按名称合并；单位为件、精度 0。
- 旧 `stored` 形成数量 1 的期初批次，旧 `taken` 成为零库存历史。
- 旧自由文本位置全部进入根仓“待分配”叶位，原文永久保存在迁移映射。
- 缺失成本须会计组三人中至少两票和经理批准前禁止取出或进入定稿报表。
- 已被旧程序物理删除且不在备份中的数据不得伪造恢复，必须报告为不可恢复。
- 迁移使用预检、单条原子 DDL、事务化 DML 回填、切换和独立清理；失败拒绝启动并保留真实状态，不声称多条 DDL 整体回滚。

## 目标数据表族

本节固定职责和关键约束，不表示表已经存在。最终字段和索引必须在所属阶段接口冻结时与本规范逐项核对。

- 身份与治理：用户、身份历史、权限、TOTP、恢复码、数据库会话、管理员范围、员工分配/搭档、暂停、临时代管、治理申请/决定、通知。
- 组织与目录：`warehouses`、`storage_locations`、品类、货物主数据、旧货迁移映射。
- 库存事实：批次、流水、批次分配、`inventory_balance`、归档计时。
- 申请：`stock_operation_requests`、三种类型明细、审批、员工入库复核、附件关联、来源建议、预留、余量取消、幂等结果和执行结果。
- 附件：内容、不可变版本、业务关联、隔离状态和扫描证据。
- 会计：分录、借贷行、原币/CNY、锁定汇率、会计组/成员、议程/表决、调整申请、核对与报告状态、账册 outbox。
- 跨仓：调拨、双边审批、发运、在途批次、收货、差异、双边证据、隔离库存和解决结果。
- 审计与集成：`audit_chain_head`、`audit_events`、版本化外部映射、投递尝试、外部确认、服务账户、范围、凭据指纹、轮换/撤销和重放记录。
- 历史事实默认禁止级联删除；可更新投影、当前状态和不可变事实必须在数据库与应用层清晰区分。

## 目标 API 资源边界

所有新接口计划位于 `/api/v1`，当前尚未实现。旧 `/api/login` 与 `/api/goods*` 冻结保留到对应迁移、对账和用户审批完成。

- 会话、TOTP、重新认证和通知。
- 仓库树、仓内位置树、品类、SKU、汇总库存及货位/批次展开。
- 隔离附件上传、扫描状态、版本和安全下载。
- 仓库上下文内的库存申请及 `approve`、`reject`、`withdraw`、`reconsider`、`cancel-remainder`、`execute` 命令。
- 治理、暂停恢复、临时代管、季度复核和警告。
- 会计分录、账册、报表、调整、议程和表决。
- 跨仓申请、双边审批、发运、收货、差异证据和解决。
- 账册事件、确认、服务账户和外部映射。

普通列表使用稳定键游标，默认 50、最大 200，响应包含 `items` 和 `nextCursor`。错误对象包含 `code`、`message`、`details` 和 `requestId`。浏览器写请求使用 Cookie、`X-CSRF-Token`、Origin/Fetch Metadata；服务接口使用独立 mTLS/HMAC 路径。
