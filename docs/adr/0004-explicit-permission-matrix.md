# 使用显式权限矩阵代替角色等级继承

仓库员工、管理员、经理、监察员、会计和代理商是职责不同的身份，而不是一条可安全继承权限的等级链。系统对每项受保护业务动作检查明确权限，并为各身份配置权限集合；每个账户同一时刻只能拥有一种身份，职责范围、临时代管和暂停另行建模。因而会计或监察员不会因为虚构的角色等级而获得库存操作或系统管理能力，“管理员及以上”等规则也改写为具体身份拥有审批权限；身份变更保留历史并立即撤销旧会话。

多仓模式下，管理员的有效职责范围由一个品类与一个仓库子树求交得到；同一品类可以由不同管理员分别负责互不重叠的仓库子树，但有效范围不得重叠。仓库员工再限定到具体仓库及获分配货物。权限判断必须同时检查动作权限、品类、仓库树范围、货物分配、临时代管和暂停状态，不能只检查身份名称或单个 `warehouse_id`。经理、监察员和总会计具有全部仓库范围，普通会计的财务职责仍按品类划分并可按仓库筛选。
