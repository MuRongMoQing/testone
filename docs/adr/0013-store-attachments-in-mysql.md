# 首版将附件内容存入 MySQL

业务单据、代理合同和审计附件使用独立附件表保存二进制内容，并同时记录原文件名、MIME 类型、字节大小、SHA-256、上传者和上传时间；单文件上限为 20 MiB。只接受 JPEG、PNG、WebP、PDF 和 DOCX，并同时校验扩展名、MIME 与真实文件签名；拒绝加密文件、旧版 DOC、含宏 DOCM、SVG、可执行文件和压缩包，下载时强制使用附件模式。附件一旦关联业务记录便不可覆盖，替换通过新增版本完成。该方案让业务记录、文件内容及版本关系在同一备份和恢复边界内保持一致，接受数据库体积增长；未来可在保持附件接口与标识不变的情况下迁移至对象存储。

管理员依据 ADR 0006 可以在季度内后补单据，但后补附件仍作为隔离的新版本保存，只有通过安全审查后才计为已补齐。暂时未上传附件不等于可以用风险文件、扫描异常文件或未完成扫描的文件满足单据要求。
