# 生产附件扫描采用失败关闭

上传附件先存入隔离状态，通过扩展名、MIME、真实签名及加密和宏限制后，再将二进制流交给 ClamAV 扫描；只有明确判定安全的版本才能被计为有效业务单据或合同。检出风险时永久隔离并警告上传者、经理和监察员；扫描服务不可用、扫描报错或签名库超过 48 小时未更新时同样保持隔离并阻止要求逐笔附件的业务提交或生效。系统保存病毒名称、引擎版本、签名版本、扫描时间和结果，不自动删除风险文件，以安全与审计证据优先于附件流程可用性。

管理员依据 ADR 0006 获准季度后补单据的操作，不以执行当时已经存在附件为生效前提；其后补上传的文件仍保持隔离，只有明确扫描安全后才计为已补齐。除此例外外，要求逐笔单据的员工申请、执行和代理合同，在安全附件版本关联前不得提交或生效。该例外不允许任何未扫描或扫描异常文件进入有效状态。
