# 高权限身份强制使用 TOTP 双因素认证

管理员、经理、监察员和会计首次进入业务界面前必须绑定 TOTP，仓库员工和代理商可选启用；TOTP 密钥加密保存，恢复码只保存哈希。身份变更、密码重置或管理员权限恢复会撤销全部旧会话但不解除既有 TOTP，高权限账户丢失第二因素时必须通过匹配其身份的双人恢复流程重建。该设计接受额外登录与恢复成本，以降低密码泄露导致库存、治理或财务权限被接管的风险。

首版以 OpenSSL 3 实现 RFC 6238，使用兼容主流认证器的 HMAC-SHA1 TOTP；libsodium XChaCha20-Poly1305 加密保存 TOTP 密钥。主加密密钥由可替换密钥存储端口从部署挂载文件读取，环境变量只保存文件路径，密钥值不得进入数据库、源码、日志或审计载荷。

高权限账户执行账户治理、管理员权限恢复、临时代管、仓库结构变更、重大会计决定或外部账册映射时，最近一次 TOTP 验证不得超过 10 分钟；超过时须完成敏感操作重新认证。重新认证成功后轮换会话标识并记录验证时间，但不扩大身份或责任范围。
