# 旧货数据迁移不猜测 SKU 合并

每条旧 `goods` 记录生成独立的 `LEGACY-Gxxxxxx` SKU，默认单位为件、数量精度为零，并进入待分类品类；不按名称猜测合并。所有旧 `goods.location` 自由文本统一迁入根仓库的“待分配”叶级货位，原始文本永久保存在旧货迁移映射中，之后由经理通过审计式流程分配正式货位，不能自动把自由文本建成正式位置。

旧 `stored` 行迁移为数量一的期初批次并保留入库时间和操作员，旧 `taken` 行作为零库存历史归档。缺失进价标记为期初成本待确认，在会计组三人中至少两人同意并经经理批准前禁止取出，也不进入定稿财务报表；后续归类、货位分配或重分类必须保留审计历史。已经被旧程序物理删除且不在备份中的数据不能伪造恢复，迁移报告必须明确列为不可恢复。
