# 审计记录采用追加表和哈希链

审批、执行、权限、配置、附件、会计和复核动作必须与对应业务变更在同一事务中追加 `audit_events`，记录操作者、当时身份与职责范围、动作、目标、关联申请、原因、前后摘要、IP、User-Agent 和时间。应用数据库账号仅可查询和插入，触发器拒绝更新和删除。

首版所有审计事件形成一条全局链。事件字段使用带版本号的长度前缀规范字节编码，以 libsodium SHA-256 计算哈希；每条事件包含前一事件哈希。业务事务完成申请、预留、余额和批次等业务锁定后，追加审计前锁定唯一 `audit_chain_head` 行；同一事务内多条事件按确定顺序连续计算并在提交前更新链头。校验器报告首个断链位置，以检测历史删除、插入或改写。

审计表用于追踪和校验，各业务表仍保存当前状态，系统不因此改为完整事件溯源架构。全局链头会串行化审计追加，这是为获得单一全局顺序而接受的取舍，未经新决策不得改为分仓或分月多链。
