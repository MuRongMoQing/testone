# 全新货物提案先占用 SKU 再等待首次入库

提交全新货物申请时，系统立即在货物主表建立 `PENDING_APPROVAL` 记录并占用唯一 SKU，以防并发提案或后续货物重复使用同一业务身份；此时不创建库存余额，也不向普通用户显示为在用货物。申请获批后，货物品种进入 `APPROVED_AWAITING_FIRST_RECEIPT` 状态，只有首次入库实际执行成功后才创建库存批次与余额并转为 `ACTIVE`。

提案被撤回或最终拒绝时仍永久保留，其 SKU 不得重新分配给其他货物。申请人修改被拒提案时创建同一复核链上的新不可变申请版本，不覆盖原提案、附件或审批结果。该生命周期以少量不可复用编码为代价，避免 SKU 在审计历史中代表不同货物。
