# 区分待审批撤回与获批余量取消

申请人在最终审批前可以立即撤回自己的申请，系统保留原申请、附件、撤回动作和审计记录，不产生库存或会计事实。获批申请不得直接撤回或修改；申请人只能创建独立的剩余量取消申请，并沿用原申请的管理员优先、经理一小时超时接管流程。对应管理员暂停时由经理直接处理取消申请。

取消获批时，只关闭尚未执行的数量并释放对应库存预留。尚无执行结果的原申请关闭为未执行取消；已有部分执行结果时关闭为部分执行后取消。此前每次执行形成的库存流水、批次分配、余额变化和会计分录均保持不可变，不能借取消流程冲正。执行期限届满时系统无需额外审批，自动取消未执行余量并释放预留。

全新货物提案即使在审批前撤回，其货物主记录和审计历史仍保留，已经占用的 SKU 不得复用。该区分让申请人可以停止尚未发生的业务，同时不把撤回或取消误用为删除历史及逆转已经发生的库存事实。
