# 管理员职责范围取品类与仓库子树的交集

每名管理员的有效范围由恰好一个 `category_id` 与一个 `warehouse_root_id` 共同确定，并包含该根节点的全部后代仓库。初始单仓部署时所有管理员绑定唯一根仓库，表现为一名管理员负责一个品类全部货物；首版目标数据库、领域和 API 已按真实多仓模型设计，新增仓库并完成组织分配后，不同管理员可以负责同一品类下互不重叠的仓库子树，而无需增加身份或复制品类。

同一品类的两个有效管理员范围不得存在祖先、后代或相同根节点关系，否则同一申请会出现多个主管。系统在范围分配事务中利用仓库层级关系检查重叠并拒绝冲突；范围变更保留有效期和审计历史。临时代管仍是单独、限货物且限时的授权，不修改永久范围分配。

仓库员工绑定一个具体仓库、一个上级管理员和明确货物集合，操作范围是员工仓库、管理员品类及货物分配的交集；员工搭档必须处于同一仓库和同一管理员范围。经理与监察员覆盖全部仓库和品类，总会计覆盖全部仓库和品类财务数据；普通会计组继续按两个品类负责财务工作，但所有查询与报表可以按仓库过滤。

每个授权判断必须组合显式动作权限、品类范围、仓库子树、货物分配、临时代管和权限暂停状态。该设计增加范围计算和重叠校验，但避免把多仓权限重新塞回身份等级或为每个仓库复制角色。
