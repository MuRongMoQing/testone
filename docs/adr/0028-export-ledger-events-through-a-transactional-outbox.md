# 通过事务性发件箱导出账册事件

复式库存会计子账是库存与成本分录的权威事实源。每次分录成功入账时，系统在同一 MySQL 事务中向事务性发件箱追加不可变账册导出事件；事务回滚时分录和事件一并回滚。导出事件具有稳定唯一标识、接口版本和顺序游标，可由外部总账、上级仓库账册或下级仓库账册增量拉取，也可由可替换适配器异步推送并安全重试。

规范事件至少包含分录及借贷行标识、来源业务与执行标识、科目、原币金额、人民币金额、锁定汇率、入账时间、仓库 ID、入账时仓库层级路径快照、冲正或补充分录关联及完整性摘要。上级账册按仓库子树聚合相同事实，下级账册按具体仓库过滤；系统不为不同层级复制或重新生成会计分录。历史层级归属使用入账时路径快照，当前管理视图可另外按现行仓库树查询。

外部系统可以幂等回传接收状态、外部凭证号、失败代码和处理时间，这些确认作为独立记录追加，不能修改库存、流水、内部科目或原分录。外部发现差错时只能发起内部会计调整流程，待受控反向或补充分录入账后再产生新的导出事件。核心领域只依赖账册导出端口，不绑定特定总账供应商、传输协议或外部编码。

外部系统通过独立的不可交互服务账户访问这些接口。账户按仓库子树、品类和动作最小授权，由经理发起、监察员批准，不复用六类人员账户或浏览器会话；外部接收确认能力不包含修改内部事实的权限。传输认证、签名、重放防护、密钥保存和撤销规则由 ADR 0035 统一约束。

内部科目、仓库和币种到外部编码的映射采用带生效区间的版本化配置，历史事件保留所用映射版本。建立或改变映射可能改变外部账册含义，按重大会计决定处理，须由对应三人会计组至少两人同意；任何非全票结果及后续回退遵守既有会计治理规则。该设计以发件箱存储和异步状态管理换取分录与集成消息不丢失，并阻止外部账册成为绕过内部审计的改账入口。
