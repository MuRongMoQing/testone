# 以 RAII、预处理语句和有界连接池封装 libmysql

系统保留当前 `libmysql` 依赖，不引入 ORM 或让 C API 继续散布在路由和业务代码中。`warehouse_mysql` 作为唯一 MySQL 适配模块，封装库生命周期、工作线程上下文、有界连接池、连接租约、预处理语句、结果集和事务；模块外不得持有 `MYSQL*`、`MYSQL_STMT*` 或依赖 MySQL 头文件。

每个应用工作单元从连接池独占一条连接，并用该连接创建本事务内的全部仓储。应用服务决定提交，未提交事务在 RAII 析构时回滚；仓储不得自行提交或把连接保存到工作单元之外。池容量与 HTTP 有界线程池和数据库部署容量共同配置，连接不能跨线程同时使用，失效或状态不洁净的连接不得归还池中。

所有业务值通过 `MYSQL_STMT` 参数绑定，删除 `mysql_real_escape_string()` 加字符串拼接的业务查询。动态排序、列名和其他不能参数化的 SQL 标识符只允许从编译期或代码内允许列表选择。金额、数量与汇率使用定点值或规范十进制字符串映射 MySQL `DECIMAL`，不经过 `double`；附件二进制使用预处理长数据接口。版本化迁移是受版本控制的可信固定 SQL，由启动迁移器在业务流量开始前单独执行。

MySQL 客户端库由进程级 RAII 对象初始化和结束，每个执行数据库工作的线程使用线程级 RAII 上下文。会话统一设置 UTC、严格 SQL 模式并关闭静默自动重连，工作单元按用途显式选择隔离级别。应用层只对已识别的死锁或锁等待超时进行有界重试，并沿用原幂等键；其他错误立即回滚和上报。该选择需要编写一层小型类型安全封装，但保留现有跨平台依赖并消除全局连接、SQL 注入和事务泄漏边界。
