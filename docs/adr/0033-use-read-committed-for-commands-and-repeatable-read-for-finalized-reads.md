# 写命令使用 READ COMMITTED，定稿读取使用一致性快照

审批、预留、入库、取出、移位、跨仓调拨和会计入账等短时读写工作单元统一使用 `READ COMMITTED`。应用服务对将要判断并改变的申请、预留、库存余额和批次使用 `SELECT ... FOR UPDATE` 或等价写入锁，并按申请、预留、余额、批次的固定顺序获取锁；普通非锁定读取不能作为库存或审批并发保护。唯一约束、检查约束和受影响行数在提交前提供最后一道数据库校验。

定稿报表、账册导出批次和审计校验使用有执行时限的 `REPEATABLE READ, READ ONLY, WITH CONSISTENT SNAPSHOT`，确保同一输出中的多条查询来自一致视图。普通列表与详情查询使用短时读取，不跨分页请求或用户会话保持数据库快照。长报表应通过后台任务或只读副本扩展，而不是让在线写库长期保留旧版本。

`SKIP LOCKED` 仅允许用于发件箱和通知等队列领取，因为工作项可由任一消费者处理且稍后重试；它不得用于可用库存、FIFO 批次、审批候选或会计判断。系统不全局启用 `SERIALIZABLE`，关键不变量由显式行锁、稳定锁顺序、数据库约束、幂等键和有界死锁重试共同保护。该组合要求测试真实并发路径，但减少不必要的范围锁与长报表对在线业务的影响。
