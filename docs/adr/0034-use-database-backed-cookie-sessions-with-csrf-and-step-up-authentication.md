# 使用数据库 Cookie 会话、CSRF 防护和敏感操作重新认证

系统删除当前进程内、可预测且无到期时间的 Bearer Token。浏览器登录改用 libsodium 生成的 256 位随机不透明令牌，MySQL 只保存令牌哈希及会话元数据；生产浏览器通过 `__Host-warehouse_session` Cookie 携带，设置 `Secure; HttpOnly; SameSite=Strict; Path=/` 且不设置 `Domain`。令牌不得进入 `localStorage`、URL、应用日志或审计载荷。

管理员、经理、监察员和会计会话由服务端在闲置 15 分钟或创建满 8 小时后失效；仓库员工和代理商在闲置 30 分钟或创建满 12 小时后失效。登录、身份变化、密码重置、TOTP 恢复和管理员权限恢复轮换或撤销相关会话。数据库持久化让多实例和服务重启共享撤销与到期事实，不允许客户端时间延长会话。

账户治理、管理员权限恢复、临时代管、仓库结构变更、重大会计决定和外部账册映射要求高权限用户最近 10 分钟内重新验证 TOTP。重新认证成功会轮换会话标识并更新最近验证时间；它只确认当前操作者，不授予新权限，也不替代双人或三人审批。

所有依赖浏览器 Cookie 的 POST、PUT、PATCH 和 DELETE 请求必须携带与会话绑定的不可预测 CSRF Token 自定义头，并校验 Origin 或 Fetch Metadata；SameSite 只提供纵深防御。安全方法不得改变服务端状态，CORS 只允许部署配置中的明确来源。外部系统集成使用独立服务凭据，不复用浏览器 Cookie。该设计增加会话表与前端 CSRF 协调，但阻止服务重启绕过撤销、脚本读取令牌和跨站伪造高风险操作。
