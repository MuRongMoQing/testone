# 为账册集成使用受限服务账户

外部总账及上下级仓库账册不复用人员账户、浏览器 Cookie 或六类人员身份。每个对接方使用独立、不可交互的服务账户，授权明确限定到仓库子树、品类以及账册事件拉取、推送接收或接收确认动作；它只能读取导出事件或追加外部确认和引用号，不能修改库存、流水、分录、外部映射或审计事实。

服务账户由经理发起创建、授权、轮换或撤销，并须由监察员批准后生效。账户元数据、凭据指纹、范围、有效期、使用、失败、越权、轮换和撤销均被审计；私钥和共享密钥通过可替换密钥存储端口从部署挂载文件读取，环境变量只保存文件路径，不以明文写入数据库、源码、日志或审计载荷。疑似泄露时立即撤销，外部账册映射仍独立遵守会计组三人中至少两人同意的重大决定流程。

所有集成通信强制 HTTPS；账册拉取与接收确认优先使用双向 TLS。服务证书和 HMAC 密钥最长有效 30 天，新旧凭据最多重叠 12 小时，可按部署条件增加 IP 允许列表。异步推送还使用 HMAC-SHA256 签名事件 ID、时间戳和原始请求正文；接收方在解析载荷前验证签名，以恒定时间比较摘要，只接受服务器时间正负 2 分钟内的时间戳，并依据事件唯一标识拒绝重放请求。

该设计增加服务账户治理、密钥存储和证书轮换成本，但把机器访问与人员会话分离，限制单个凭据泄露的影响范围，并使外部接收确认不能成为绕过内部会计与审计控制的改账入口。
