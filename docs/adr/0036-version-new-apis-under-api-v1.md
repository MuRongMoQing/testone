# 新业务接口统一使用 `/api/v1`

所有计划中的新业务接口统一位于 `/api/v1`。当前 `/api/login`、`/api/goods` 和 `/api/goods/take` 作为冻结的迁移兼容契约，在对应新用例、数据迁移、对账及用户审批完成前保留，但不再增加新能力；最终退役是独立、可审查的切换动作。

普通列表使用稳定键游标，默认返回 50 条、最大 200 条，响应包含 `items` 和 `nextCursor`。统一错误对象包含 `code`、`message`、`details` 和 `requestId`。审批决定和每次库存执行通过 `Idempotency-Key` 传递已确认的幂等键。该选择增加新旧接口并行期，但避免在原路径上进行不可验证的破坏性替换。
