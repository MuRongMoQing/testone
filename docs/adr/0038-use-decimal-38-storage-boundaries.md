# 使用 `DECIMAL(38, scale)` 保存库存与金额

库存数量使用 `DECIMAL(38,6)`，每个 SKU 声明 0 至 6 位允许精度；单位进价和售价使用 `DECIMAL(38,4)`，汇率使用 `DECIMAL(38,8)`，原币和人民币总额使用 `DECIMAL(38,2)`。JPY 业务金额必须为整数，人民币、USD 和 EUR 总额保留两位并采用四舍五入。

每次数量、进价和售价必须大于零，超精度、溢出或超出可用库存时拒绝命令。应用使用定点值或规范十进制字符串与 MySQL 交换数据，不经过 `double`。该边界以较大数据库字段换取统一的跨模块精度和可预测的溢出检查。
